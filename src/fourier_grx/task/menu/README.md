# Task Menu ID 设计说明

## 为什么不同机器人的 Task ID 可以重复？

这是**有意为之的设计**，目的是保持上位机（PC / App / SDK）接口的一致性。

### 设计原则

- 相同功能语义的任务（如"站立控制"、"步行"）在不同机器人型号上使用**相同的 ID 值**。
- 上位机发送任务指令时，只需关心"我要执行什么动作"，而不需要感知当前连接的是哪个机器人型号。
- 底层根据已注册的机器人型号路由到对应的具体 Task 实现。

### 示例

```python
# fi_task_menu_grmini1.py
TASK_STAND_CONTROL = 3010   # GRMini1 的站立控制

# fi_task_menu_grmini3.py
TASK_STAND_CONTROL = 3010   # GRMini3 的站立控制（相同 ID，不同实现）
```

上位机统一发送 `3010`，系统根据当前机器人型号自动调度正确的 Task。

### 注意事项

- **同一机器人型号内**的 Task ID 必须唯一，不能重复。
- **跨机器人型号**的 ID 重复是允许的，但语义应保持一致（相同 ID 应代表相同的功能意图）。
- 新增机器人型号时，优先复用已有 ID 以保持接口一致；确实需要新功能时再分配新 ID。
